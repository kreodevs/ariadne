Ariadne Architecture

AriadneSpecs — Servicios, datos y flujos de interacción

Flujo Principal

  • • Frontend → API → Ingest → FalkorDB
  • • Repos remotos sincronizan hacia Ingest
  • • Ingest indexa código en FalkorDB + Postgres
  • • Chat/Analysis fluye Ingest ↔ Orchestrator
  • • MCP expone herramientas a la IA (Cursor, etc.)

Almacenamiento

  • • Postgres: repos, sync_jobs, usuarios, credenciales
  • • FalkorDB: grafo de código (nodos, edges, embeddings)
  • • Redis: cola BullMQ (sync asíncrono) + caché LRU
  • • Sharding: por proyecto (FALKOR_SHARD_BY_PROJECT)

Comunicación Interna

  • • Red Docker falkor-net — todos los servicios
  • • API proxy a Ingest (quita prefijo /api)
  • • Ingest delega LangGraph al Orchestrator
  • • Orchestrator consulta grafo vía API → FalkorDB
  • • MCP habla directo a FalkorDB e Ingest

AriadneSpecs • docker-compose • red interna falkor-net • Sin puertos expuestos (excepto dokploy-network)
